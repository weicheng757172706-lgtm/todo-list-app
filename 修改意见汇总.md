## 一、界面与布局

### 1.1 表格形式
- 任务栏使用 Excel 风格的表格形式（Treeview 控件）
- 表头使用专业蓝色主题，支持横向和纵向滚动条
- 行颜色根据优先级自动标记

### 1.2 字体调整
- 表头字体：14号 → 12号（缩小两号）
- 表格内容：12号
- 表格行高：45px
- 标题：20号加粗
- 标签/输入：13号
- 按钮：12号加粗

### 1.3 布局自适应
- 输入区域从单行改为两行自适应布局（pack() 嵌套 Frame）
  - 第1行：[任务内容标签] + [输入框(stretch)] + [添加按钮]
  - 第2行：[任务紧急程度] + [任务类别] + [责任人]
- 窗口初始 900x650，最小 700x500
- "任务内容"列设为 stretch，缩放时优先伸缩
- 不使用 Grid 跨行列布局（避免列隐藏 bug）

### 1.4 按钮调整
- 移除两个"🔄 刷新"按钮（冗余）
- "重要紧急程度" → "任务紧急程度"
- "删除记录" → "删除所选"

### 1.5 标题修改
- 窗口标题 + 页面蓝色标题：`Todo List vX.X - 每日任务管理`
- 第一次修改：`ToDoList（w.c.）vX.X`
- 第二次修改：去掉版本号 → `ToDoList（w.c.）`
- 第三次修改：去掉 w.c. → `ToDoList`

## 二、核心功能

### 2.1 任务字段
- 任务内容（文本框）
- 任务紧急程度（4 级 Eisenhower 矩阵下拉框）
  - 重要紧急 / 重要不紧急 / 不重要紧急 / 不重要不紧急
- 任务类别（下拉框）
  - 每日成长任务 / 工作任务
- 责任人（文本框，支持多人，以 `,，;；、|/` 分隔）
- 创建时间（自动记录）
- 完成时间（标记完成时自动记录）

### 2.2 ID 显示逻辑
- 待完成/已完成列表各自从 1 开始排显示行号
- 删除/移走任务后，后续任务的显示 ID 自动前移填补
- 内部保留永久 ID 用于数据定位（Treeview iid 隐藏传递）

### 2.3 添加任务
- 填写内容、选择紧急程度、选择类别、填写责任人 → 点击"✚ 添加任务"
- 添加成功后清空输入框，状态栏显示"任务已添加"
- **无成功弹窗 toast**

### 2.4 标记完成
- 选中待完成任务 → 点击"✓ 标记完成"
- 移入已完成列表，自动记录完成时间
- **无成功弹窗 toast**

### 2.5 编辑任务（v5.8 新增）
- 待完成任务操作栏「✏ 编辑」按钮（橙色）
- 弹出编辑窗口，支持修改：任务内容、任务类别、任务紧急程度、责任人
- 内容不能为空验证，保存后状态栏显示"任务已更新"

### 2.6 删除功能
- **待完成任务**：「🗑 删除任务」— 删除选中的待完成任务
- **已完成任务 - 删除所选**：「🗑 删除所选」— 删除选中的已完成记录
- **已完成任务 - 删除全部**：「🗑 删除全部」（v5.7 新增）— 一键清空所有已完成记录
- 所有删除操作**无确认弹窗 / 无成功 toast**

### 2.7 导出 Excel（v6.4 新增）
- 已完成任务「📤 导出 Excel」按钮（蓝色）
- 弹出"另存为"对话框，默认文件名：`已完成任务_YYYYMMDD.xlsx`
- 导出全部已完成记录
- 格式化：蓝色表头+白色加粗字体、数据行边框、自动列宽、冻结首行

## 八、Bug 修复记录

### 8.1 v3 — 启动崩溃
- 原因：`tk.Entry` 不支持 `height` 参数，`ttk.Treeview` 列不支持 `font` 参数
- 修复：移除 height 参数，用 style 设置字体

### 8.2 v5.5 — 启动崩溃
- 原因：`format_person()` 静态方法内 `return` 后代码缩进错误导致字体属性未初始化
- 修复：将代码移入 `__init__`，删除残缺的旧 `__init__`

### 8.3 v6.0 exe 无法打开
- 原因：旧 build 缓存污染（`.spec` / `build/` 未彻底清理）
- 修复：`rm -rf build dist *.spec` 后重打包 + 添加 `--hidden-import tkinter`
